# 欢迎页面 (WelcomePage)

`WelcomePage` 是应用的第一入口，集成品牌展示、初始化逻辑与隐私合规拦截。

## 核心职责

* **品牌展示**：利用 `flutter_animate` 衔接 Logo 动效，建立第一视觉印象。
* **合规拦截**：**强制性**隐私协议弹窗。未同意前严禁调用任何敏感 API 或 SDK。
* **业务初始化**：集中处理鉴权检查、第三方 SDK 加载，并确保异常时仍能降级启动。
* **路由控制**：初始化完成后，使用 `pushReplacement` 销毁欢迎页并跳转至首页。

## 逻辑流程

1.  **入场**：播放 Logo 动画。
2.  **检测**：动画结束，检查持久化状态 `isPrivacyAgreed`。
    * **未同意**：弹出 `CuBottomSheet` 协议窗口。
    * **已同意**：直接执行 `_appInit`。
3.  **启动**：执行插件初始化 -> 跳转主页。

## 开发规范

### 1. 性能要求
* **异步并行**：使用 `Future.wait` 初始化非依赖性插件，缩短白屏时间。
* **任务分级**：非核心插件（如埋点、日志）建议延迟到首页加载后初始化。

### 2. 合规标准
* **不可规避**：协议弹窗须设置 `isDismissible: false`，确保用户必须做出选择。
* **二次确认**：用户拒绝时须弹出 `CuConfirm` 告知风险，并提供安全退出路径。

### 3. UI 准则
* **语义化文案**：所有文本必须通过 `context.l10n` 调取，严禁硬编码。
* **沉浸式体验**：使用 `backgroundColor` 适配品牌色，确保与系统状态栏完美融合。
